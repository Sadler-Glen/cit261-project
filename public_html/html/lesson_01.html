Topic 01: Loops, Conditional Statements, Functions, Variables, Parameters, Arrays, Associative Arrays

Before we tackle Loops, Conditional Statements, Arrays and Associative Arrays it is probably prudent that we explain variables and functions in the first instance.

Variables & Functions

The variables in Javascript can be thought of as being named containers. You place data in the container and then refer to the container by naming the container. Before you use a variable in a program you must declare or initialize it. This is done with the var keyword as follows.

Declaring a variable

var x;
var name;

Initializing a Variable

var name = "Glen"
var name;
name = "Glen";

Javascript Variable names

  • Do not use Javascript reserved keywords, i.e., do not use break or boolean as variable names as they are not valid.
  • Javascript variable names cannot start with a number (0-9). They must begin with a letter or and underscore character. Eg., 59check is invalid, however _59check is a valid variable name.
  • Javascript variable names are also case sensitive. For example Name and name are two different variables.

Functions & Parameters

A Javascript function is a block of code designed to complete a particular task and is executed or runs when called or invoked.

For example:

  • When an event occurs such as when a user clicks on a button.
  • When it is called from some Javascript code.
  • Or it may call itself and therefore run automatically.

The function is defined with the function keyword, followed by a name, followed by parenthesis ().

Function names follow the same rules as variable names and can contain numbers, letters, and dollar signs.

Parameters may be included in the parenthesis and the code that is to be run is within the curly braces {},as per the example below.
function name(parameter1, parameter2, parameter3,...) {
code to be executed
return value;
}

Function return

When the return statement is reached the function will stop running. If the function was called from a statement, Javascript will return to the calling statement and start running the code after the calling statement. If the function calculates a return value, this value is returned back to the calling function.
The beauty of functions is that once they have been defined or created, they can be reused many times with different parameters or arguments to produce different results.

The following is example code which calculates all the odd integers from 1 to n (user input). The function is named sumOdd() and has no parameters passed to it.

What the function does

  1. The user is prompted for a positive integer.
  2. The function is invoked.
  3. The input is validated.
  4. The for loop starting at 1 will check for odd numbers up to the inputted integer and sum these numbers with the use of the modulus feature built into Javascript.
    function sumOdd() {
        // covert the input to an integer
        var n = parseInt(document.getElementById("numberBox").value);

        var sum = 0;
         //check for 0 or -ve numbers
        if (n <= 0) {
            return document.getElementById('outputDiv').innerHTML =
                    "You must enter a Positive Integer";
        }
        // sum the odd numbers from 1 to n (user input) 
        for (var i = 1; i <= n; i++) {
            if (i % 2 != 0) {
                sum += i;
            }
        }
        // Output the sum to the outputDiv in the webpage
        document.getElementById('outputDiv').innerHTML =
                "The sum of all odd integers from 1 to " + n + " is " + sum;
    }

Loops

There are several kinds of loops utilised in Javascript which essentially end up doing the same thing. They are used to repeat an action a number of times depending on the requirements of the function that it is implemented in. Our example Loop below repeats the action "n" times i.e., the number that is input by the user.

Notable other loops that are implemented in Javascript are:

  • do...while loop
  • while loop
  • for...in loop

for loop

The for loop executes until a condition evaluates to false and has the following form.

for (initial expression; condition; increment expression;){
statement's
}

do...while Loop

The do...while loop, like the for loop, repeats until a specified condition is false. The difference between the two is that the statement executes once in the do...while loop before the condition is checked. This is because the while condition is at the end of the code. See the syntax below.

do
statement's
while (condition);

while loop

The while loop repeats for as long as the specified condition is true. Unlike the For and do...while loops, the while loop tests the condition prior to the statements are executed. As soon as the condition evaluates to false, the loop is exited and the statement following the while loop takes control. See the syntax below.

while (condition)
statement's

for...in loop

The for...in loop, loops through the properties of an object and for each property executes the statements. See the syntax below.

for (variable in object) {
statement's
}

Conditional Statements

As in every other programming language, the code in Javascript needs to make decisions based on varying conditions, for example, something as simple as, depending on the weather, what footwear should I wear?

There are a number of conditional statements used in Javascript and they are listed below.

  • if
  • if...else
  • switch

Logical Operators: AND, OR and NOT

Logical operators can help miniise the need to use nested if...else statements.

  • &&(AND)- all expressions must evaluate to be true for the whole expression to be true.
  • ||(OR) - One or more expressions must evaluate to be true for the whole expression to return true.
  • !(NOT) - This operator is used to negate an expression.

Switch

The switch statement becomes valuable when you want to take a single expression or value as an input and then look through a number of choices until one matches, then execute the corresponding code.

Arrays

The Array is an object that is used to store multiple values in a single object.
Each array value can easily be accessed individually. You can create an array by declaring it, initialising it or with the use of the new keyword.

Ways to create / declare an Array

    var motorcycle = [];
    var motorcycle = ['Honda','Suzuki','Yamaha','Indian'];
    motorcycle = new Array('Honda','Suzuki','Yamaha','Indian');

Arrays are super flexible and you can do so many things with an array. The following basic things can be achieved with an array. Lets use the array that we declared above as an example.

var motorcycle = ['Honda','Suzuki','Yamaha','Indian'];

Things we can do with the array (Array Methods)

  • Add to or remove from the end of an array.

    • motorcycle.push('Ducati'); - add to the end of the array.
    • motorcycle.pop('Indian'); - remove from the end of the array.

  • Add to or Remove from the beginning of the array.

    • motorcycle.shift('Ducati'); - add to the beginning of the array.
    • motorcycle.unshift('Honda'); - remove from the beginning of the array.

  • Search and find elements in an array.
  • Retrieve the number of elements in the array. motorcycle.length

There are a plethora of other things that can be done with arrays.

Associative Arrays

An Associative array in javascript is an object and can be declared or created dynamically. It is in essence a set of key/value pairs as you see in our pseudocode example below. This means that the value is stored in "association" with its key - hence the name of the array. Curly braces are used as opposed to square brackets as you see below.

var arr = {key1 : 'value1', key2: 'value2', key3 : 'value3'};
The key may be a string, a number or an identifier. Lets look at an example that we can use to show how to access and use Associative arrays. We need to remember that objects are sometimes referred to as associative arrays. Let's say we have a car object as below.
var car = {make: "Toyota", model: "Corona", year: 2017};
We can access the properties of the above car object as below.
car["make"] which returns "Toyota"
car["model"] which returns "Corona"
car["year"] which returns 2017
A simple for loop cannot be used to scan the elements in an Associative Array because the elements are not accessible by an index, so it is much simpler and more ideal to use the for...in loop. The beauty of the for...in loop is that you do not need to the position of the last element. Here is a piece of code that shows the use of the for..in loop 
    var carInfo = document.getElementById("carDiv");
    var btn = document.getElementById("btn");
    btn.addEventListener("click", function () {
        var car = {make: "Toyota", model: "Corona", year: 2017,color: "Green"};
        writeInfo(car);
        btn.classList.add("hide-btn");
    });

    function writeInfo(car){
        var htmlString = "";
        for (var key in car) {
            htmlString += '<p>' + car[key];
        }
        carInfo.insertAdjacentHTML('beforeend', htmlString);
    }
